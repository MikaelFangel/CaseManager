      API Reference Case Manager v0.1.0

      Modules

      CaseManager

      CaseManager keeps the contexts that define your domain and business logic.

      CaseManager.Alerts

      Domain that defines resources related to alerts and defines the allowed routes for the JSON API.

      CaseManager.Alerts.Alert

      Resource that represents an alert in the system. The Resource is to be used by the JSON API and also to view and edit alerts within the application.

      CaseManager.AppConfig

      Domain that controls everything related to app configuration.

      CaseManager.AppConfig.File

      Resource for file uploads.

      CaseManager.AppConfig.Setting

      Resource used to store application wide settings in a persitent manner.

      CaseManager.Cases

      Domain to represent a case and its relations.

      CaseManager.Cases.Case

      Resource that represents a single case in the system.

      CaseManager.Cases.Comment

      Resource that represents a comment belonging to a case in the system.

      CaseManager.Cases.File

      Resource for file uploads.

      CaseManager.Policies.CaseOwnerPolicy

      Implementation of the SimpleCheck behaviour. This policy checks if a given case and user team_id is the same.

      CaseManager.Policies.MSSPCreatePolicy

      Implementation of the SimpleCheck behaviour. This policy checks if a user is part of the mssp team and if so it passes.

      CaseManager.Relationships

      Domain that represents cases and their related resources.

      CaseManager.Relationships.CaseAlert

      Resource for the many-to-many relationship between cases and alerts.

      CaseManager.Release

      Used for executing DB release tasks when run in production without Mix installed.

      CaseManager.Repo
      CaseManager.SelectedAlerts

      GenServer that holds the state of which alerts a user has selected. This is only meant to store data for the mssp users.

      CaseManager.Teams

      Domain that reprensents teams and their related resources.

      CaseManager.Teams.Email

      Resource that represents emails. This resources is different from the email used for authentication.

      CaseManager.Teams.IP

      Resource to represents an IP address of either version v4 og v6.

      CaseManager.Teams.Phone

      Resource that represents a phone number.

      CaseManager.Teams.Secrets

      Helper module to fetch the secret needed for Ash authentication to work.

      CaseManager.Teams.Team

      Resource for managing teams withing the application. A Team is supposed to be an entity that are used to group users.

      CaseManager.Teams.Token

      Resource representing a user token.

      CaseManager.Teams.User

      Resource representing a user in the system.
